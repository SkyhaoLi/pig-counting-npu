# 多元目标泛化场景与微调方案

本目录归档本系统的五大可泛化场景，每个场景包含：
- 数据需求
- 模型微调要点
- 输出字段扩展
- 4 周落地路线

详细技术路线请参考 `competition/猪只智能感知与健康预警系统_作品设计草稿.docx` 第六章。

---

## 场景 A:肉牛 / 奶牛健康监测

| 项目 | 内容 |
|------|------|
| 数据需求 | 500-1000 段牛只视频，BCS（1-5 级）人工标注 |
| 检测微调 | YOLOv8m（目标更大），fine-tune 200 epoch |
| 体重模型 | MobileViT-S 输出层重训为 BCS 回归 + 体重双头 |
| 世界模型 | V-JEPA 2 LoRA：反刍行为时间序列特化 |
| 新输出字段 | `BCS_score` / `rumination_minutes` / `lying_time_ratio` |
| 周期 | W1 数据采集 → W2 检测器训练 → W3 多任务微调 → W4 现场验证 |

## 场景 B:肉鸡 / 蛋鸡场密集小目标

| 项目 | 内容 |
|------|------|
| 数据需求 | 鸡舍俯拍视频（密度高、目标小） |
| 检测微调 | 输入分辨率 640→1024，Anchor 重聚类，NMS 收紧到 0.3 |
| 追踪调整 | Tracking 关联距离 30px（默认 100px），track_buffer=120 |
| 新模块 | 密度热力图（基于网格统计），死鸡识别（姿态长期不变） |
| 新输出字段 | `density_heatmap.png` / `dead_bird_count` / `egg_box_count_daily` |
| 目标指标 | IDF1 ≥0.80，死鸡误报 ≤1 / 日 |

## 场景 C:水产渔业（网箱 / 塘口）

| 项目 | 内容 |
|------|------|
| 数据需求 | 水下 RGB-D 视频 + 红外补光夜视 |
| 增强 | UWGAN 水下增强，对抗光线衰减 |
| 检测微调 | Underwater-YOLO（融合 hazy/clean 损失） |
| 追踪调整 | ByteTrack 关联代价加入光线衰减置信度修正 |
| 行为微调 | V-JEPA 微调"游动轨迹熵 + 浮头"两类异常 |
| 新输出字段 | `school_density` / `stress_index` / `surface_oxygen_alert` |

## 场景 D:实验动物（小鼠 / 兔）行为分析

| 项目 | 内容 |
|------|------|
| 数据需求 | 直接复用 Animal-JEPA 已开放的 MB3 数据集（IEEE 2025） |
| 模型 | V-JEPA 2 LoRA 微调，目标 9 类行为 Top-1 ≥0.94 |
| 业务接口 | 对接药理学时间窗（dosing time），生成药效响应曲线 |
| 新输出字段 | `experiment_label` / `drug_response_curve.json` / `behavior_class_per_min` |
| 周期 | 1 周复用数据 + 1 周微调 + 1 周对接 + 1 周对照实验 |

## 场景 E:公共安全与工业计数（跨域）

| 项目 | 内容 |
|------|------|
| 检测器替换 | YOLOv8-Person（人流） / YOLOv8-SKU（流水线） |
| 框架复用 | 三线计数 + ByteTrack 完全保留 |
| 新模块 | 踩踏预警（密度梯度突变） / SKU 漏件检测 |
| 应用场景 | 地铁通道客流统计、商场漏报警情、流水线计件、机场行李分拣 |
| 周期 | W1-2 检测器替换 + W3 异常检测 + W4 现场测试 |

## 复用矩阵

| 模块 | 是否复用 | 备注 |
|------|---------|------|
| ByteTrack 跟踪器 | ✓ | 任意场景通用 |
| 三线中位数计数 | ✓ | 跨场景核心算法 |
| 跳帧 + Kalman 占位 | ✓ | 通用优化 |
| ZoneAnalyzer 状态机 | ✓ | 只需调 zone 划分参数 |
| YOLOv8 检测头 | ✗ | 必须按品种 / 物体重训 |
| 体重估计 | 部分 | 哺乳动物可复用，禽类需重设计 |
| V-JEPA 行为头 | ✓ | LoRA 微调，主干冻结 |
| Web 监控 UI | ✓ | CSS 主题可换 |

## 实施信心声明（评委可信）

* 场景 D（实验动物）的可行性已由 Animal-JEPA（IEEE 2025）独立验证。
* 场景 A、B 的微调路线基于本系统验证过的训练代码（`competition/training/`）。
* 场景 C、E 的训练脚本待数据到位后 1-2 周可补齐。
* 受比赛筹备周期所限，赛前 demo 仅落地生猪场景；其余四场景**有完整技术方案 + 训练代码就绪**。
